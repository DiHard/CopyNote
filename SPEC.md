# CopyNote — спецификация

Этот документ описывает, **что** должна делать утилита, без привязки к конкретному технологическому стеку. Его можно использовать как техническое задание для реализации на любом подходящем стеке.

## 1. Суть продукта

CopyNote — лёгкая десктопная утилита для Windows, в которую пользователь заносит короткие текстовые «заметки» (ярлык + значение, например «Личная почта» → `name@example.com`). Клик по заметке копирует её значение в системный буфер обмена.

Проект будет выложен в open source под лицензией **MIT**, бесплатно. Автор: **dihard**. Идентификатор приложения: **dev.copynote.app**.

## 2. Целевая аудитория и сценарий

Пользователь часто копирует одни и те же короткие строки (адреса, логины, номера телефонов, реквизиты, сниппеты) и хочет иметь их под рукой в одном окне, где любую из них можно получить в буфер обмена одним кликом.

## 3. Платформа

Только **Windows 10/11**. Кросс-платформенность не требуется и не является целью.

## 4. Приоритеты (в порядке важности)

1. **Скорость работы.** Холодный старт приложения — менее 300 мс. Появление окна из трея — менее 50 мс. Копирование при клике — мгновенное, без заметной задержки.
2. **Малое потребление ресурсов.**
   - Оперативная память в простое: < 50 МБ.
   - Размер инсталлятора: < 15 МБ (желательно ~5 МБ).
   - Нулевая фоновая активность при бездействии (никаких таймеров, пуллингов, сетевых запросов).
3. **Красивый современный интерфейс.** Минималистичный, аккуратный, с поддержкой светлой и тёмной темы по системным настройкам. Приятные анимации появления/наведения без визуального «шума».

## 5. Функциональные требования (MVP)

### 5.1. Управление записями (CRUD)
- Создание новой записи: ярлык (обязателен, ≥ 1 символ после trim) + значение (может быть пустым или многострочным).
- Редактирование существующей записи.
- Удаление записи с предварительным подтверждением.
- Отображение списка всех записей.

### 5.2. Копирование в буфер обмена
- По клику на запись её значение помещается в системный буфер обмена.
- После успешного копирования — визуальный фидбек:
  - короткая отметка «Copied» на самой карточке записи, и/или
  - всплывающий toast `"<label>" copied` в нижней части окна, исчезающий через ~1.5 секунды.
- Если копирование не удалось — показать toast с ошибкой (тон «ошибка»).

### 5.3. Поиск
- Поле поиска в шапке окна.
- Фильтрация в реальном времени по совпадению подстроки (без учёта регистра) в поле `label` ИЛИ `value`.
- Если под поиск ничего не попадает — показывается сообщение «Nothing matches your search».

### 5.4. Пустое состояние
- Если записей нет — в центре области списка показывается подсказка с кнопкой «Add your first entry».

### 5.5. Системный трей
- При запуске приложения в трее появляется иконка с тултипом `CopyNote`.
- Контекстное меню иконки содержит пункты:
  - **Open CopyNote** — показывает и фокусирует главное окно.
  - **Quit** — завершает работу приложения.
- Клик левой кнопкой по иконке трея — показывает/фокусирует главное окно.
- Закрытие главного окна (крестиком) **не завершает приложение**, а сворачивает его в трей. Завершение — только через пункт «Quit» в меню трея.

### 5.6. Хранение данных
- Все данные хранятся **локально**, в файле `%APPDATA%\CopyNote\data.json`.
- Формат хранения — человекочитаемый JSON (см. раздел 7).
- Шифрование **не применяется** — сценарий предполагает не чувствительные данные (почта, публичные реквизиты и т. п.).
- Запись в файл должна быть **атомарной**: сначала записать во временный файл, затем переименовать в целевой, чтобы исключить повреждение при сбое питания / kill процесса.
- При первом запуске или отсутствии файла — стартовать с пустым стором, не падать.

## 6. Нефункциональные требования

- **Бинарный размер** итогового исполняемого файла/инсталлятора — как можно меньше.
- **Нулевая фоновая активность** в простое (без тиков таймеров, сетевых коннектов, опроса файлов).
- **Нет сетевых соединений вообще.** Приложение полностью офлайновое. Никакой телеметрии, аналитики, автообновлений по сети.
- **Нет внешних зависимостей рантайма**, которые пользователь должен ставить вручную после установки (или они уже есть в Windows 10/11 из коробки).
- **Тёмная/светлая тема** по системному предпочтению (`prefers-color-scheme`) — определяется автоматически, без настроек.
- **Графика и иконки** — без сторонних шрифтов/CDN, всё локально.

## 7. Контракт данных

Файл `data.json`:

```json
{
  "version": 1,
  "entries": [
    {
      "id": "c2b9e5a7-3f1d-4a28-9a21-8f1b9e3a7c11",
      "label": "Личная почта",
      "value": "name@example.com",
      "order": 0,
      "createdAt": "2026-04-08T12:00:00Z",
      "updatedAt": "2026-04-08T12:00:00Z"
    }
  ]
}
```

Поля:
- `version` *(integer)* — номер схемы, для будущих миграций формата. Стартовое значение `1`.
- `entries` *(array)* — массив записей.
- `entries[].id` *(string, UUID v4)* — уникальный идентификатор записи, генерируется при создании.
- `entries[].label` *(string)* — отображаемый ярлык, после `trim` не пустой.
- `entries[].value` *(string)* — значение, копируемое в буфер обмена. Может быть пустой строкой или содержать переносы строк.
- `entries[].order` *(integer, ≥ 0)* — позиция в списке. Меньшие значения — выше. После удаления — перенумеровываются подряд без «дыр».
- `entries[].createdAt`, `entries[].updatedAt` *(string, ISO 8601 UTC)* — временные метки создания и последнего изменения.

## 8. Бизнес-логика

- **Создание записи.** Вставляется в начало списка (`order = 0`), все существующие записи сдвигаются вниз (`order += 1`). `createdAt` и `updatedAt` = текущий момент.
- **Обновление записи.** Меняются только `label` и `value`; `order` сохраняется; `updatedAt` обновляется на текущий момент.
- **Удаление записи.** После удаления оставшиеся записи сортируются по `order` и их `order` переназначается как `0, 1, 2, …` (упаковка без дыр).
- **Сортировка отображения.** Перед показом список всегда сортируется по `order` по возрастанию.
- **Валидация `label`.** После `trim` не должен быть пустым. Попытка создать/обновить запись с пустым `label` — ошибка с понятным сообщением.

## 9. UI / UX

### 9.1. Главное окно
- Компактное, ориентировано вертикально. Рекомендуемый стартовый размер: **420×640**, минимум **360×480**.
- Окно открывается по центру экрана при первом запуске.
- Структура сверху вниз:
  1. Шапка: иконка-логотип + название «CopyNote» + кнопка «+ New» справа.
  2. Поле поиска.
  3. Прокручиваемый список карточек-записей.

### 9.2. Карточка записи
- На карточке видны: `label` (крупнее, жирным) и `value` (мельче, приглушённым цветом, обрезается многоточием).
- Весь основной блок карточки — кликабельная область «скопировать».
- Справа, при наведении на карточку, появляются две иконки-кнопки: «Edit» (карандаш) и «Delete» (корзина).
- После успешного копирования на карточке кратковременно (~900 мс) показывается бейдж «Copied».

### 9.3. Модалка создания/редактирования
- Полупрозрачный фон за окном.
- Два поля: `Label` (обычный input) и `Value` (textarea, растяжимая по вертикали).
- Кнопки «Cancel» и «Create» / «Save».
- Фокус автоматически в поле `Label`.
- `Esc` — закрывает модалку без сохранения.
- Кнопка сохранения неактивна, пока `label` пуст (после trim).

### 9.4. Подтверждение удаления
- Отдельная модалка с вопросом `Delete entry?`, подтекстом `"<label>" will be permanently removed.`, кнопками «Cancel» и «Delete».

### 9.5. Toasts
- Всплывают снизу по центру окна.
- Два типа: `success` (зелёный) и `error` (красный).
- Живут ~1.5 секунды, затем плавно исчезают.
- Не блокируют ввод.

### 9.6. Темы
- Цвета подбираются автоматически под `prefers-color-scheme: light | dark`.
- В тёмной теме — тёмно-синий градиент фона, светлые карточки с лёгкой прозрачностью.
- В светлой теме — мягкий светло-серый градиент фона, белые карточки.

### 9.7. Общее
- Все пользовательские тексты интерфейса — **на английском** (label, placeholder, сообщения, toasts, меню трея). Локализация не требуется в MVP.
- Шрифт — системный sans-serif (Inter / Segoe UI / system-ui).

## 10. Внешнее API / интеграция

Никакого внешнего API у приложения нет. Никаких сетевых вызовов, никакой интеграции с облаком, почтой, мессенджерами и т. п.

## 11. Что НЕ входит в MVP

Явно исключено, чтобы не раздувать scope:
- Облачная синхронизация между устройствами.
- Шифрование содержимого (данные не считаются чувствительными).
- Менеджер паролей / MFA / генерация токенов.
- История буфера обмена, мониторинг буфера.
- Глобальный горячий клавишный шорткат для показа окна (явно убран из MVP на старте).
- Drag & drop сортировка списка (может быть добавлена позже — поле `order` уже спроектировано под это).
- Импорт/экспорт.
- Автозапуск при старте Windows.
- Мультиязычность интерфейса.
- Любая телеметрия и автообновление по сети.

## 12. Распространение

- Windows installer (MSI или EXE) размером < 15 МБ.
- Исходники и релизы — на GitHub, лицензия MIT.
- README с кратким описанием, скриншотом, инструкцией по сборке из исходников и ссылкой на скачивание.

## 13. План реализации (пошагово)

Независимо от выбранного стека, рекомендуется разбить реализацию на следующие инкременты. Каждый шаг должен приводить к работоспособному состоянию приложения.

1. **Scaffold.** Создать пустой проект выбранного стека, запустить «Hello world» окно.
2. **Модель и хранилище.** Реализовать структуру `Entry` и `Store`, функции `load` / `save` с атомарной записью в `%APPDATA%\CopyNote\data.json`. Покрыть unit-тестами (round-trip, пустой стор, отсутствующий файл).
3. **Слой бизнес-операций.** Реализовать операции `list`, `create`, `update`, `delete`, `reorder` над стором, с валидацией и персистом после каждой мутации.
4. **Минимальный UI списка.** Отобразить список существующих записей, сделать кнопку «New» и простую форму создания (без стилей).
5. **Копирование в буфер.** Сделать клик по записи → запись значения в системный буфер обмена + временный индикатор «Copied».
6. **Редактирование и удаление.** Модалка редактирования, модалка подтверждения удаления.
7. **Поиск.** Поле поиска + фильтрация списка.
8. **Toasts.** Всплывающие уведомления успеха/ошибки.
9. **Трей.** Иконка в трее, меню (Open / Quit), перехват закрытия окна → сворачивание в трей.
10. **Стили и темы.** Приведение интерфейса к целевому виду (карточки, hover, светлая/тёмная тема, шрифты, иконки, анимации).
11. **Иконка приложения.** Подготовить иконки нужных размеров (ico / png) и подключить в окно и в инсталлятор.
12. **README и LICENSE.** Оформить документацию, добавить скриншот, указать лицензию MIT.
13. **Сборка релиза.** Собрать инсталлятор, замерить размер, RAM в простое, время холодного старта. Убедиться, что укладываемся в целевые значения из раздела 4.

## 14. Критерии приёмки MVP

- Можно создать, отредактировать, удалить запись.
- Клик по записи копирует её значение в буфер обмена, в любом стороннем приложении `Ctrl+V` вставляет это значение.
- Поиск фильтрует список по подстроке в `label` или `value`.
- Данные сохраняются в `%APPDATA%\CopyNote\data.json` и корректно загружаются при следующем запуске.
- При закрытии окна приложение уходит в трей; через меню трея можно открыть окно снова или завершить приложение.
- Инсталлятор < 15 МБ, RAM в простое < 50 МБ, холодный старт < 300 мс.
- Приложение работает в обеих темах (светлая и тёмная) без визуальных артефактов.
- Нет сетевых соединений (можно проверить сниффером — трафика быть не должно).
